# Build Log: v1.4.1 — volume OSD feature-check guards

**Date:** 2026-04-24 | **Mode:** BUILD (fix) | **Design doc:** none — single-commit targeted fix

---

## Origin

Triggered mid-session after shipping v1.4.0 (upstream v0.72.0 merge) when smoke-testing on UC3. User reported a volume OSD showing in the Kodi media browser even though their `mmadalone/integration-kodi-patch` fork's `suppress_volume_overlay` toggle (patch 6, commit `d8cb60e`, v1.18.13-madalone.1) was enabled.

Investigation revealed the root cause: the Kodi fork's toggle correctly removes `Features.VOLUME`/`VOLUME_UP_DOWN`/`MUTE*` from the entity's feature set when enabled. But 7 of 8 `volume.start()` call sites in remote-ui don't check the entity's feature flag before firing the OSD — they unconditionally call `entityObj.volumeUp()` (no-op if feature removed) followed by `volume.start(entityObj)` (fires OSD regardless).

Only `Activity.qml` correctly gates its `VOLUME_UP`/`VOLUME_DOWN` handlers on `hasFeature(MediaPlayerFeatures.Volume_up_down)`. Every other file — `Page.qml` (home-screen fallback), `MediaBrowser.qml` (where the user hit it), and the 5 media-player deviceclass overrides (`Receiver`, `Speaker`, `Tv`, `Streaming_box`, `Set_top_box`) — bypassed the check.

**This is an upstream UC bug** present in stock UC Remote 3 firmware. Our fork's fix is upstream-contributable.

## Plan

Wrap each of the 14 unguarded call sites with the same `hasFeature(MediaPlayerFeatures.Volume_up_down)` guard that `Activity.qml` already uses. No new imports — every affected file already imports `Entity.MediaPlayer 1.0`.

Pattern:
```qml
// Before:
"VOLUME_UP": {
    "pressed": function() {
        entityObj.volumeUp();
        volume.start(entityObj);
    }
},

// After:
"VOLUME_UP": {
    "pressed": function() {
        if (entityObj.hasFeature(MediaPlayerFeatures.Volume_up_down)) {
            entityObj.volumeUp();
            volume.start(entityObj);
        }
    }
},
```

`Page.qml`'s handler references `mediaComponentEntity` instead of `entityObj` but follows the same pattern.

## Decisions made

- **Root-cause fix over workaround:** chose to fix all 7 unguarded call sites rather than add a remote-ui `Config.showVolumeOverlay` global toggle. Aligns with owner's per-device preference and the existing feature-advertisement contract. Upstream-contributable.
- **14 edits in one commit:** kept as a single atomic commit to make the upstream-PR cherry-pick clean.
- **Both `volumeUp()` and `volume.start()` inside the guard:** symmetric with `Activity.qml`'s pattern. Prevents no-op dispatch too.
