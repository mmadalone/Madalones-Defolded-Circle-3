## Decisions made

**Option B scope** ÔÇö We kept our BaseDetail.qml RowLayout (Option A from v1.3.0) and adopted upstream's naming elsewhere. Rejected upstream's inline battery `Row` in BaseTitle + Activity to avoid duplicate renders (our Loader in BaseDetail already owns the chip).

**QSettings migration default** ÔÇö Seeded `ui/batteryEveryWhere` with legacy value falling back to v1.3.0's default `true`. Users who accepted the v1.3.0 default keep the chip visible after upgrade. Fresh v1.4.0 installs also get `true` ÔÇö technically diverges from upstream's `false` default but honors ┬ğ1.3 "Never remove features" for continuity with v1.3.0 UX.

**Icon set ÔÇö accept upstream as-is** ÔÇö Initial concern (Explore agent read) that upstream removed `uc:heat`/`uc:brightness`/`uc:list`/`uc:bluetooth`/`uc:battery-low` was overblown. Upstream's JSON file had *duplicate keys* (primary definitions + aliases section). They deduplicated by removing the aliases-section copies. Grep confirmed all 5 icon names still resolve post-merge (some with different glyphs: `uc:list` `´Ç║` ÔåÆ `¯Æë`, `uc:heat` `¯Çî` ÔåÆ `´ü¡`). Zero QML patches needed.

**Ui.qml Flickable clamp** ÔÇö Upstream's value is `1100` for a `~52-line` toggle block; we had bumped to `1260` for our `~42-line` block. Upstream likely has a latent clipping bug here (smaller clamp for larger content). Kept upstream's `1100` for now, flagged for on-device verification. If clipping observed, bump clamp in a follow-up fix.

**Docker toolchain entrypoint** ÔÇö Non-obvious gotcha: `unfoldedcircle/r2-toolchain-qt-5.15.8-static:latest` has `ENTRYPOINT ["/entrypoint.sh"]` which swallowed arbitrary `bash -c "..."` commands. Had to use `--entrypoint bash` to get a proper shell for the `lupdate` invocation. Documented for future sessions.
